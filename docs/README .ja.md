# BOA Core – Documentation

このディレクトリは、  
**BOA / RCA / RP がすでに揃った状態で読むための説明資料**です。

ここに書かれているのは、実装手順でも仕様の詳細でもありません。  
この docs の目的はただ一つです。

> **判断・責任・停止が  
> どの順序で現れ、  
> どこで閉じるのかを誤解なく伝えること**

---

## この docs に含まれないもの

- 実装方法
- API / Schema の詳細
- 哲学的背景の全文
- ユースケース集

それらは、それぞれのリポジトリや仕様書にあります。

---

## この docs が扱うもの

- なぜ BOA が最初に置かれるのか
- BOA / RCA / RP の役割の違い
- 判断が「進む」「止まる」「戻る」構造
- 失敗や判断不能をどう扱う設計か
