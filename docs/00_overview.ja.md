# Overview: What This Architecture Is

BOA Core は、  
**AIが関与するシステムにおいて、  
判断・責任・停止を壊さずに扱うための構造群**です。

このリポジトリは「正しい答えを出す」ためのものではありません。  
**「判断できない状態を、正しく止める」ために存在します。**

---

## これはフレームワークではありません

BOA / IDG / RP / RCA は、
一般的な意味でのレイヤードアーキテクチャやフレームワークではありません。

- 処理を分割するための層ではない
- 責務を整理するための役割分担でもない

これらはすべて、

> **判断の流れに沿って現れる「関所」**

です。

---

## 含まれる4つの構造

この BOA Core には、以下の4つが含まれます。

- **BOA (Boundary-Oriented Architecture)**  
  判断が許される境界を宣言する

- **IDG (Interface Determinability Gate)**  
  判断可能な状態かどうかを評価する

- **RP (Resolution Protocol)**  
  判断を進める／止める／戻すことを明示的に選ぶ

- **RCA (Responsibility Closure Architecture)**  
  判断の責任を閉じ、残す

重要なのは、  
**これらが同時に存在するのではない** という点です。

---

## 判断の基本的な流れ（要約）

1. システムは BOA に到達する  
2. 境界条件を満たした場合のみ、IDG に進む  
3. IDG が「判断可能」と示した場合のみ、RP が起動する  
4. RP は必ず以下のいずれかを返す  
   - Resolve（決める）
   - Reject（却下する）
   - Return（人や上流に戻す）
5. RCA が、この判断の責任を閉じる

---

## この設計が最も重視すること

この構造が最優先するのは、成功率でも速度でもありません。

- 判断不能を曖昧にしないこと
- 「止める」という選択肢を消さないこと
- 人が戻る余地を残すこと

BOA Core は、  
**判断を自動化するための設計ではなく、  
判断を壊さないための設計**です。
